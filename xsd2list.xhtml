• service(/service)

    The app:service element is the container for service information associated with one or more Workspaces.

    • author(/service/author)

      Author

      optional

      • name(/service/author/name)

        Human-readable name for the person

        xs:string required

      • uri(/service/author/uri)

        IRI associated with the person

        xs:anyURI optional

      • email(/service/author/email)

        E-mail address

        optional

        Restriction of xs:normalizedString Pattern: \w+@(\w+\.)+\w+

    • workspace(/service/workspace)

      Workspaces are server-defined groups of Collections. The "app:workspace" element contains zero or more app:collection elements describing the Collections of Resources available for editing.

      required unbounded

      • title(/service/workspace/title)

        Human-readable title for the Workspace.

        xs:string required

        • type(/service/workspace/title/@type)

          Restriction of xs:token Enumeration: text,html,xhtml

      • collection(/service/workspace/collection)

        Describes a Collection

        optional unbounded

        • href(/service/workspace/collection/@href)

          The IRI of the Collection

          xs:anyURI

        • title(/service/workspace/collection/title)

          A human-readable title for the Collection

          xs:string required

          • type(/service/workspace/collection/title/@type)

            Restriction of xs:token Enumeration: text,html,xhtml

        • accept(/service/workspace/collection/accept)

          A media range as defined in [RFC2616]. The media range specifies a type of representation that can be POSTed to a Collection.

          xs:string optional unbounded

        • categories(/service/workspace/collection/categories)

          Provides a list of the categories that can be applied to the members of a Collection.

          optional unbounded

          • fixed(/service/workspace/collection/categories/@fixed)

            Value of either "yes" or "no", indicating whether the list of categories is a fixed or an open set. The absence of the "fixed" attribute is equivalent to the presence of a "fixed" attribute with a value of "no".

            Restriction of xs:string Enumeration: yes,no

          • category(/service/workspace/collection/categories/category)

            Provides a list of the categories that can be applied to the members of a Collection.

            optional unbounded

            • scheme(/service/workspace/collection/categories/category/@scheme)

              An IRI that identifies a categorization scheme

              xs:anyURI

            • term(/service/workspace/collection/categories/category/@term)

              The "term" attribute is a string that identifies the category

              xs:string

            • label(/service/workspace/collection/categories/category/@label)

              Provides a human-readable label for display in end-user applications.

              xs:string
